PoolCensus Desktop

View latency, payout, and job health for every public stratum endpoint directly from a modern desktop interface.

Run from source

Install Go (1.23+) and Node.js (LTS) if you haven’t already. Then:

  • npm install inside frontend/
  • npm run build in frontend/ to refresh the UI bundle
  • wails build (or wails dev while iterating)

During development you can also run wails dev, keep the browser UI open, and click the green "Start scan" button whenever you want fresh data.

Bootstrap scripts

Each platform has a helper script that installs the OS dependencies, Go/Wails, and frontend bundle before running wails build for the first time.

  • scripts/setup-ubuntu.sh – Ubuntu/Debian; installs GTK/WebKit, Node, and Go packages before rebuilding the frontend.
  • scripts/setup-mac.sh – macOS (requires Homebrew) to install Go/Node and regenerate the bundle.
  • scripts/setup-windows.bat – Windows using Chocolatey; installs Go, Node, and pkg-config and then runs the frontend build.

Run the script for your platform before invoking wails build or wails dev to reduce manual dependency work.

WebKitGTK note (Linux)

Ubuntu 24.04 ships libwebkit2gtk-4.1-dev, but Wails still asks for the 4.0 pkg-config name. scripts/setup-ubuntu.sh now creates a symlink for webkit2gtk-4.0.pc.

If you skip the bootstrap script, manually run sudo ln -sf /usr/lib/x86_64-linux-gnu/pkgconfig/webkit2gtk-4.1.pc /usr/lib/x86_64-linux-gnu/pkgconfig/webkit2gtk-4.0.pc so the build can find WebKit.

Packaging scripts

After dependencies are installed, the build helpers regenerate the frontend bundle and produce platform-specific distributions.

  • scripts/build-all.sh – convenience entrypoint; runs scripts/build-cross.sh (Linux + Windows, and macOS if osxcross is configured).
  • scripts/build-linux.sh – runs npm run build in frontend/ and then wails build for Linux.
  • scripts/build-mac.sh – equivalent packaging steps for macOS.
  • scripts/build-windows.bat – runs the frontend build and wails build from a Windows shell.
  • scripts/build-cross.sh – Ubuntu 24.04 helper that builds Linux + Windows and optionally macOS if osxcross is installed (installs mingw-w64 automatically; pass --no-install-deps or --no-darwin).
  • scripts/install-osxcross.sh – installs osxcross (requires you to supply a macOS SDK tarball you are licensed to use).

Use these scripts when you’re ready to create packaged binaries for each platform.

macOS packaging

macOS bundles still require Apple tooling. If you’re building on macOS, run scripts/build-mac.sh to produce a native app bundle.

If you want to attempt macOS builds from Ubuntu, install osxcross via scripts/install-osxcross.sh and then run scripts/build-cross.sh. The macOS portion uses -nopackage, so it won’t produce a signed/notarized .app or .dmg from Linux (it writes poolcensus-darwin-amd64/poolcensus-darwin-arm64 into build/bin).

The cross-built .app targets older Intel macOS by default (via MACOSX_DEPLOYMENT_TARGET_AMD64). If you want an Apple Silicon build, pass --macos-arm64 (requires macOS 11+).

Using the desktop app

Open PoolCensus, click Start scan, and watch the progress bar walk through every configured pool. When the scan completes the UI will show how many hosts looked healthy and which ones reported issues.

Scanning is entirely manual now; the frontend never starts a background scan without your input.

Custom pools

Drop a pools.json next to the binary to override the embedded list. Each scan will automatically prefer that file if it exists.

Want to see the raw data? Check pools.json or inspect the generated scan view while the app is open.
